Ecosistemas de Comunicación: Web APIs

Una Web API es una interfaz de programación que permite a las aplicaciones compartir datos y comunicarse mediante el protocolo HTTP. Actúa como una capa intermedia para procesar transferencias entre sistemas.

Paradigmas: REST vs SOAP

SOAP: Protocolo basado en XML, independiente de plataforma, con reglas estrictas de seguridad (WS-Security).

REST: Estilo arquitectónico ligero que utiliza URIs y formatos como JSON. Es el estándar para conectar componentes en microservicios debido a su flexibilidad y capacidad de caché.

Desarrollo y Seguridad (JWT)

Para crear una API exitosa, se deben definir objetivos claros, documentar con herramientas como Swagger y versionar la interfaz. La seguridad se gestiona mediante JSON Web Token (JWT), compuesto por Header, Payload y Signature, garantizando transmisiones íntegras y seguras.

Métodos HTTP
  • GET: Recuperar recursos.
  • POST: Crear nuevos datos.
  • PUT/PATCH: Actualizar.
  • DELETE: Eliminar registros.

Arquitectura de Microservicios

En este enfoque, una aplicación se compone de muchos servicios pequeños e independientes. Cada uno tiene su propia tecnología y base de datos, comunicándose vía APIs REST o mensajería (Kafka).

Beneficios Organizativos
  • Escalabilidad: Se escalan solo los servicios necesarios (ej. el carrito de Amazon).
  • Resiliencia: El aislamiento de fallas evita que un error detenga toda la aplicación.
  • Agilidad: Actualizaciones rápidas sin tocar el resto del monolito.

Manejo de Contenedores

Los contenedores virtualizan a nivel de Sistema Operativo. Son mucho más ligeros que las máquinas virtuales porque comparten el kernel del anfitrión.

Docker: Herramienta líder para empaquetar software en imágenes portátiles.

Kubernetes: El orquestador que coordina y programa miles de contenedores a gran escala, asegurando alta disponibilidad.
