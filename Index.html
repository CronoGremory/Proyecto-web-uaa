Ecosistemas de Comunicación: Web APIs

Una Web API es una interfaz de programación que permite a las aplicaciones compartir datos y comunicarse entre sí mediante el protocolo HTTP. Actúa como una capa de abstracción para que dos sistemas interactúen sin conocer su mecánica interna.

Ejemplo Práctico: Uber y Google Maps

Cuando usas Uber, la aplicación consume la API de Google Maps para obtener geolocalización. Del mismo modo, un e-commerce utiliza pasarelas como PayPal para gestionar cobros electrónicos seguros.

Paradigmas: REST vs SOAP

SOAP: Es un protocolo basado en XML con reglas estrictas de seguridad y transaccionalidad, ideal para el sector financiero.

REST: Arquitectura moderna que utiliza JSON como formato estándar. Es la base de las aplicaciones móviles actuales porque permite almacenamiento en caché, optimizando el rendimiento.

Seguridad y Tokens JWT

En APIs privadas, se utiliza el JSON Web Token (JWT). Es un estándar (RFC 7519) para transmitir información segura entre partes. Se compone de tres bloques fundamentales: Header, Payload y Signature.

Métodos HTTP Críticos
  • GET: Solicitar información.
  • POST: Enviar nueva información.
  • PUT: Actualizar datos existentes.
  • DELETE: Eliminar registros.

Arquitectura de Microservicios

Los microservicios proponen dividir una aplicación monolítica en piezas pequeñas e independientes.

Ejemplo: Amazon

En Amazon, el carrito de compras es un microservicio y el buscador es otro. Si el buscador falla, aún puedes pagar lo que ya tienes en el carrito. Esto los hace resilientes.

Beneficios
  • Escalado preciso por componente.
  • Equipos autónomos y multitecnología.
  • Actualizaciones rápidas sin pausas.
Desafíos
  • Dificultad de monitoreo.
  • Latencia en conexiones de red.

Virtualización Moderna: Contenedores

Los contenedores son paquetes que incluyen el código de tu aplicación y TODAS sus dependencias (bibliotecas, versiones de lenguaje). Virtualizan a nivel de Sistema Operativo, lo que los hace mucho más ligeros que las máquinas virtuales tradicionales.

Docker: Permite crear imágenes portátiles estándar.

Kubernetes: Orquestador que administra miles de contenedores, reiniciándolos si fallan y escalándolos automáticamente.
