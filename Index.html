1. Ingeniería de Web APIs: Estándares y Pasos de Creación

Una Web API es una interfaz que permite el intercambio de datos entre programas de software bajo estándares definidos, actuando como una capa de abstracción para que sistemas heterogéneos se comuniquen mediante HTTP.

Metodología Detallada para Crear una Web API:

De acuerdo con los manuales de ingeniería de software:

  1. Definición de Requerimientos: Identificar qué recursos (datos) se expondrán y los permisos de acceso de los usuarios finales.
  2. Selección del Estándar Arquitectónico: Decidir entre la flexibilidad y ligereza de REST (usando JSON) o la rigidez y robustez transaccional de SOAP (usando XML).
  3. Diseño de la Interfaz (Endpoints): Definir URIs lógicas y descriptivas que identifiquen los recursos de forma única dentro del servidor para facilitar el consumo.
  4. Capa de Seguridad (JWT): Implementar JSON Web Token, el estándar (RFC 7519) para la transmisión segura de información. El token debe contener Header, Payload y Signature para garantizar la integridad.
  5. Desarrollo y Pruebas: Codificar la lógica de negocio y realizar pruebas unitarias para validar los códigos de estado HTTP (200, 201, 404, 500).
  6. Documentación y Publicación: Exponer la API mediante herramientas como Swagger para permitir que terceros la integren fácilmente.
SOAP vs REST: Un Análisis Profundo

SOAP: Es un protocolo basado en XML independiente de la plataforma. Ofrece estándares estrictos como WS-Security, lo que lo hace ideal para transacciones bancarias.

REST: Es una arquitectura que utiliza JSON y permite el uso de caché para mejorar drásticamente el rendimiento de red. Es la opción preferida para aplicaciones móviles y sitios web modernos.

Métodos HTTP Estándar
  • GET: Solicitar y recuperar información del servidor.
  • POST: Enviar datos para crear un nuevo recurso.
  • PUT: Actualizar un recurso existente de forma completa.
  • DELETE: Eliminar un recurso específico del servidor.

2. Arquitectura de Microservicios

Los microservicios representan un cambio de paradigma hacia el desarrollo nativo de la nube. En lugar de un monolito pesado, la aplicación se divide en servicios pequeños e independientes.

Beneficios Organizativos y de Negocio:
  • Despliegue Independiente: Permite actualizaciones continuas (CI/CD) sin detener todo el sistema.
  • Escalado Horizontal Selectivo: Se escalan solo los servicios con mayor carga, optimizando el uso de CPU y memoria.
  • Resiliencia de Software: El aislamiento de fallas garantiza que un error en un componente no provoque una caída total.

3. Virtualización de Sistemas Operativos

Los contenedores constituyen un mecanismo de empaquetado lógico que abstrae las aplicaciones del entorno real. A diferencia de las máquinas virtuales (VM), los contenedores virtualizan a nivel de sistema operativo, compartiendo el kernel del anfitrión.

Ventajas sobre las VMs Tradicionales:
  • Son extremadamente ligeros y se inician en fracciones de segundo.
  • Facilitan la portabilidad: "Si corre en mi contenedor, corre en la nube".
  • Permiten una mayor densidad de aplicaciones por servidor físico.

4. Ecosistema .NET y Desarrollo Cloud

ASP.NET Core y el Patrón MVC

Es la plataforma multiplataforma de alto rendimiento diseñada por Microsoft para la nube y contenedores. Utiliza el patrón MVC para separar responsabilidades entre Modelo (datos), Vista (interfaz) y Controlador (lógica), facilitando el mantenimiento a largo plazo.

Despliegue y Orquestación Cloud

Uso de servicios como Amazon ECS y EKS para gestionar infraestructuras de contenedores sin la necesidad de administrar servidores físicos directamente. Esto habilita el despliegue en la nube híbrida con flexibilidad y portabilidad total entre proveedores como Azure y AWS.

5. Punto 5: Ecosistema Docker y Flujo de Trabajo

Docker es una tecnología de tiempo de ejecución que empaqueta software en unidades estandarizadas llamadas contenedores. Incluyen bibliotecas, herramientas de sistema y código, eliminando el problema de "en mi computadora sí funcionaba".

Operaciones Principales

Uso de Docker Build para crear imágenes inmutables y Docker Desktop para administrar el ciclo de vida del desarrollo local.

Distribución: Docker Hub

Plataforma para encontrar y compartir imágenes de contenedores, permitiendo el uso de software preconfigurado de forma inmediata.

Orquestación con K8s

Mientras Docker gestiona el contenedor individual, Kubernetes orquestra clústeres masivos asegurando el escalado automático y la alta disponibilidad.

Estructura de Seguridad JWT

Un **JWT** se compone de tres partes codificadas en Base64:

  • Header: Algoritmo de firma.
  • Payload: Información del usuario y permisos.
  • Signature: Firma digital para asegurar la integridad.
Retos de los Microservicios
  • Mayor complejidad operativa en la gestión de red distribuida.
  • Latencia latente por comunicación entre microservicios.
  • Dificultad en el monitoreo y trazabilidad de peticiones.
Docker vs Kubernetes

Docker Desktop: Administra contenedores individuales de forma aislada en desarrollo.

Kubernetes: Orquesta aplicaciones de nivel de producción a escala masiva.
